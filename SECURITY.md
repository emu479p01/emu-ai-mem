# Security Policy

## Supported versions

Security fixes are provided for the latest tagged release.

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for this repository. Do not open a
public issue containing credentials, private memory content, or an exploitable proof of
concept.

emu-ai-mem relies on Git and the hosting provider for authentication and authorization.
The `personal` and `team` metadata fields are descriptive; repository access controls are
the security boundary. Plugins run local commands with the user's privileges and should
only be installed from a trusted source.
